# AI-Assisted Phishing Email Analyser

I built this tool because every SOC job description I read mentioned 
the same problem: alert fatigue. Tier 1 analysts spend a significant 
part of their day manually triaging phishing emails, writing the same 
types of reports, over and over. I wanted to understand that workflow 
deeply, and I wanted to build something that actually addressed it.

So I built a Python tool that reads a phishing email, sends it to an 
AI model, and returns a structured incident report. It is not a 
replacement for a human analyst. It is what I would want on my screen 
if I was handling 50 alerts a shift.

## What it does

- Reads raw email samples (.txt files)
- Sends content to the Claude API with a SOC analyst prompt
- Outputs a professional incident report in Markdown including:
  - Severity rating
  - Indicators of Compromise (IOCs)
  - Phishing techniques identified
  - MITRE ATT&CK mapping
  - Recommended actions for the analyst

## How to run

1. Add email samples as .txt files to the /samples folder
2. Add your Anthropic API key to analyser.py
3. Run: python3 analyser.py
4. Reports are saved automatically to the /reports folder
